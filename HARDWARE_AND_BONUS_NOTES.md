# 硬件申请与创新加分备注

## 已确认的加分方向

1. **自训练数据集与YOLO模型**：公开数据预训练，拿到房屋模型后补拍真实门、窗、灯具和风扇，重新微调并部署到Orange Pi。
2. **节能创新包**：能耗估算、环境光自动照明、无人节能模式合并为一个完整加分点。
3. **大模型API**：仅作为联网时可启用的自然语言查询、日志总结和节能建议，不参与开门或实时控制。

## 网络硬件申请备注

建议申请：

- Orange Pi AIpro 20T及配套电源、eMMC/存储卡、散热器。
- 至少一个可用千兆或更高速以太网口、网线。
- 若板载无线不可用，申请USB Wi-Fi网卡；它是可选项，不应成为基础功能依赖。
- USB摄像头或与板卡匹配的MIPI摄像头。

需要向老师确认：

- 验收现场是否允许电脑与Orange Pi使用网线直连。
- 现场有线网络是否能访问互联网，还是只能建立本地连接。
- 是否允许Windows开启Internet Connection Sharing，将Wi-Fi互联网共享给Orange Pi的有线网口。
- 是否提供交换机、路由器、网线和固定IP配置。
- 大模型API演示能否使用预录视频作为无互联网时的备选证据。

## 连接方案优先级

### A. 离线验收，首选

```text
Arduino UNO --USB串口--> Windows电脑
Orange Pi   --网线直连--> Windows电脑
浏览器      --localhost--> Pixel Home
```

所有门禁、温控、YOLO、数据库和GUI均可离线工作。

### B. 有线联网

Orange Pi连接现场路由器或交换机，通过有线网络访问大模型API。基础系统仍保持离线能力。

### C. 电脑共享网络

电脑通过Wi-Fi上网，并通过Windows网络共享将互联网转发到Orange Pi有线网口。该方案依赖现场权限和系统配置，不作为唯一演示方案。

### D. USB Wi-Fi网卡

只有确认驱动兼容Orange Pi系统镜像后才使用。需要提前安装驱动并在验收前离线测试。

## 已预留的软件边界

- `NetworkCapability`：描述离线、有线、无线和互联网可用性。
- `DisabledLLMProvider`：未来替换为具体云端或本地模型提供商。
- `EnergyPolicy`：未来实现人员存在检测、自动照明和能耗估算。
- `/api/capabilities`：GUI和验收脚本可查询当前可用能力。
- 所有可选功能默认关闭，断网不会影响核心系统。

## 拿到硬件后的修改清单

1. 检查网口、Wi-Fi、摄像头、串口设备名和系统版本。
2. 设置Orange Pi与电脑的静态IP并测试双向通信。
3. 决定后端运行在Windows还是Orange Pi。
4. 替换串口、摄像头、YOLO和网络adapter。
5. 使用最终模型照片补充数据集并微调YOLO。
6. 测试断网、网线拔出、Arduino重连和API超时。
7. 只有现场联网稳定时才启用大模型API。
